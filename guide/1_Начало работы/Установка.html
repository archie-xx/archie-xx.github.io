# Подробная инструкция

Новичкам...

Перед первой установкой необходимо попросить тимлида или его приближенных выдать права во всех репозиториях, связанных с админ. панелью. Начиная с 3-го пункта они уже пригодятся)

Перед началом...

Согласуйте порты, названия и пароли баз данных, директории и прочее если ставите админку в боевом режиме, а не для разработки.
В любом случае, ниже приведена инструкция, которая приведет к установке рабочей версии админ. панели, конфигурацию которой можно будет поправить позже)

# Окружение

  1. Создаем пустую директорию админки, к примеру mint-admin-develop. Переходим в нее

  2. Создаем файл docker-compose.yml, в который закидываем следующий код:

DOCKER-COMPOSE.YML 
  version: '3'
  
  services:
    mysqldb:
      container_name: mint_admin_develop_mysql
      image: mysql:8
      restart: always
      environment:
        - MYSQL_ROOT_PASSWORD=gEH8iDlZyTvq
        - MYSQL_DATABASE=main
        - MYSQL_USER=main
        - MYSQL_PASSWORD=Rxa5DJD4StO3
      volumes:
        - ./docker/mysql:/var/lib/mysql
  
    adminer:
      container_name: mint_admin_develop_adminer
      image: adminer:latest
      restart: always
      ports:
        - 8090:8080
      depends_on:
        - mysqldb
  
    redisdb:
      container_name: mint_admin_develop_redis
      image: redis
      restart: always
      volumes:
        - ./docker/redis:/data
  
    php-fpm:
      container_name: mint_admin_develop_php
      image: westside072/php-fpm:7.4.12
      restart: always
      volumes:
        - ./www:/var/www
        - C:/Users/Timofei/.ssh/id_rsa:/root/.ssh/id_rsa
      working_dir: /var/www # в процессе установки будет сказано, что с этим делать
      # working_dir: /var/www/laravel 
      # command: php artisan serve --host 0.0.0.0 --port=3000
      ports:
        - 1337:3000
      depends_on:
        - mysqldb
        - redisdb
  
    horizon:
      container_name: mint_admin_develop_horizon
      image: westside072/php-fpm:7.4.12
      volumes:
        - ./www/laravel:/var/www/laravel
      working_dir: /var/www/laravel
      command: php artisan horizon
      depends_on:
        - mysqldb
        - redisdb
  
    node_chat:
      container_name: chat_node_chat
      image: node:14.9.0
      working_dir: /var/www/sockets
      command: npm run start:install
      restart: always
      ports:
        - 9000:3000
      volumes:
        - ./www/sockets:/var/www/sockets
      depends_on:
        - redisdb
  1. Клонируем пакет с чатами в www/sockets командой:
git clone git@bitbucket.org:mintrocket/mint-chats-js.git www/sockets
  1. Поднимаем контейнеры командой:
docker-compose up -d
  1. Заходим в терминал php-контейнера и устанавливаем Laravel 7 версии:
docker-compose exec php-fpm bash
composer create-project --prefer-dist laravel/laravel:^7.0 laravel

Для Linux

После установки рекурсивно раздаем права на директории bootstrap и storage в laravel:

chmod 777 -R ./laravel/bootstrap
chmod 777 -R ./laravel/storage
  1. Генерируем ssh-ключ, в контейнер закидываем id_rsa. Например для root пользователя положить его в /root/.ssh/id_rsa

Опять для Linux

Дайте права 700 на id_rsa внутри контейнера

chmod 700 /root/.ssh/id_rsa

А теперь назад в прошлое...

Останавливаем все процессы и выходим из контейнера:

exit

Останавливаем все сервисы:

docker-compose down

Заходим в файл docker-compose.yml и делаем действия с комментариями, а именно:

  • удаляем строку, которую сказано удалить;
  • находим 2 закомментированные строчки и раскоментируем их.

Возвращаемся обратно как ни в чем не бывало:

docker-compose up -d
docker-compose exec php-fpm bash

# Laravel

  1. В composer.json добавляем наш репозиторий:
"repositories": [
    {
        "type": "vcs",
        "url": "git@bitbucket.org:mintrocket/mint-admin-php.git"
    }
]
  1. Устанавливаем наш пакет с админкой:
composer require mintrocket/mint_admin "^5.0"
  1. Сконфигурировать laravel-mix, чтобы он искал фронтенд в нужной папке командой:
php artisan mint_admin:webpack

# Настройка .env

  1. Сконфигурировать подключение к БД в .env

TIP

Тут пригодятся данные, которые мы писали во втором пункте в файл docker-compose.yml, а именно этот кусочек:

environment:
    - MYSQL_ROOT_PASSWORD=gEH8iDlZyTvq
    - MYSQL_DATABASE=main
    - MYSQL_USER=main
    - MYSQL_PASSWORD=Rxa5DJD4StO3
  1. Присвоить любой APP_NAME
  2. Прописать MIX_API_HOST (в нашем окружении это http://localhost:1337/)
  3. Добавить SCOUT_DRIVER=tntsearch

# Запуск скрипта установки

  1. Запустить установку
php artisan mint_admin:setup

Во время ожидания...

Это самый долгий этап установки (минут 5, зависит от интернета и ПК), поэтому не стесняйтесь во время ожидания принять ванну, или выпить чашечку кофе 😃

# Последние шаги

  1. Добавить в конец config/scout.php данный кусок:
'tntsearch' => [
    'storage' => storage_path('framework/indexes'),
    'fuzziness' => env('TNTSEARCH_FUZZINESS', true),
    'fuzzy' => [
        'prefix_length' => 10,
        'max_expansions' => 50,
        'distance' => 3,
    ],
    'asYouType' => false,
    'searchBoolean' => env('TNTSEARCH_BOOLEAN', false),
]
  1. Импортировать индексы моделей:
php artisan index:fresh
  1. Просканировать новые файлы:
composer dump-autoload
  1. Собрать frontend:
npm run dev
