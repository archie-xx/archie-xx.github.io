# Сортировка

Для указания сортировки данных на уровне эластика нужно добавить константу SORTING и $mapping в нужную модель:

App/Models/ModelName.php


public const SORTING = [
    'SQL' => [
        'name'
    ],
    'ELASTIC' => [
        'id' => '',
        'title' => '.keyword',
        'langs'=> '',
        'town_id'=> ''
    ],  
];

protected $mapping = [
    'properties' => [
        'id' => [
            "type" => "long"
        ],
        "title" => [
            "type" => "text",
            'fields' => [
                'keyword' => [
                    'type' => 'keyword'
                ],
            ]
        ],
        "langs" => [
            "type" => "text"
        ],
        'town_id' => [
            "type" => "long"
        ],
    ],
];

Ключ SQL принимает поля из БД по которым сортировать

Нас интересует ключ ELASTIC, он принимает ассоциативный массив, где ключ это название поля, а значение это тип поля

!!! Взависимости от типа поля зависит и способ сортировки

Например, есть поля типа text - эластик проводит их анализ, сортировка по таким полям делается очень плохо. К примеру город "г. Тобольск" он начнет сортировать на букву Т, а город "г. Новый Уренгой" вообще на У, тк новый - это прилагательное. Для решения этого кейса в базу пихали поля типа keyword - этот тип не анализируется и хранит записи как есть.

Подробнее о типах полей: https://www.elastic.co/guide/en/elasticsearch/reference/current/mapping-types.html
