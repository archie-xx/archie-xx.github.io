# CRUDController

Немного теории:

CRUD:

  • Create
  • Read
  • Update
  • Delete

Четыре базовые функции при работе с базой данных, здесь эти методы реализованы и еще немного больше, чем просто CRUD

Каждый базовый метод CRUDController'a имеет свои hook-методы, которые можно и рекомендуется юзать, если необходимо ввести правки в работу метода, ежели переопределять сам метод т.к. они содержат формреквест, проверку прав и вызов хуков, которые могут потеряться при переопределении

Http/Controllers/CRUDController имеет методы:

# all

# hook_before_all()

  • хук вызывается перед исполнением основной логики метода all()
    function hook_before_all(&$query);

# hook_after_all()

  • хук вызывается после исполнения работы метода all()
    function hook_after_all(&$paginateCollection);

# all()

# переопределять только в исключительных случаях!

  • метод, который получает все записи из БД без пагинации, он содержит вызов хуков
function all(Index $request);

# index

# hook_before_index()

  • хук метод, выполняемый перед index методом
    function hook_before_index(&$query);

# hook_before_index()

  • хук метод, исполняемый после получения пагинируемоего списка
    function hook_after_index(&$paginateCollection);

# index()

# переопределять только в исключительных случаях!

  • метод, возвращаемый пагинируемый список, чаще всего юзается для отображения табличного представления
    function index($request);

# show

# hook_before_show()

  • хук перед методом show()
    function hook_before_show(&$query, $id);

# hook_after_show()

  • а этот хук метод может что-то подправить уже в найденной коллекции метода show()
    function hook_after_show(&$model);

# show()

# переопределять только в исключительных случаях!

  • этот метод возвращает единичную коллекцию, базируясь на id ######(Переопределять только в исключительных случаях)
    function show($request, $id);

##store

# hook_before_store()

  • аналогичный предыдущим хук перед store()
    function hook_before_store(&$request);

# hook_before_store()

  • аналогичный другим хук после store()
    function hook_after_store(&$model);

###store()

# переопределять только в исключительных случаях!

  • store() метод инсертит данные в базу
    function store($request);

##update ###hook_before_update()

  • хук перед update()
    function hook_before_update(&$request, $id);

###hook_after_update()

  • хук после апдейта
    function hook_after_update(&$model);
  • метод, который обновляет данные у записи в базе

###update()

# переопределять только в исключительных случаях!

    function update($request, $id);

##destroy ###hook_before_destroy()

  • хук перед удалением
    function hook_before_destroy(&$request, &$object);

###hook_after_destroy()

  • хук после удаления
    function hook_after_destroy();

###destroy()

# переопределять только в исключительных случаях!

  • метод удаления единичной записи по id
    function destroy($request, $id);

# Валидация запросов

При необходимости валидации полей для методов необходимо создать FormRequest наследовав его от соответсвующего.


use App\Http\Requests\Admin\CRUD\DestroyRequest;
use App\Http\Requests\Admin\CRUD\IndexRequest;
use App\Http\Requests\Admin\CRUD\ShowRequest;
use App\Http\Requests\Admin\CRUD\StoreRequest;
use App\Http\Requests\Admin\CRUD\UpdateRequest;

class  ExampleUpdateRequest extends  UpdateRequest
{
 public function rules()
    {
        return [
            // правила 
        ];
    }

}

И переопределить в классе переменную, заменив Request наследником.

class ExampleController extends CRUDController
{
   // protected $showRequest = ShowRequest::class;
   // protected $indexRequest = IndexRequest::class;
   // protected $storeRequest = StoreRequest::class;
    protected $updateRequest = ExampleUpdateRequest::class;
   // protected $destroyRequest = DestroyRequest::class;
