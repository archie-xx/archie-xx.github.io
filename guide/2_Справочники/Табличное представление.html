# Добавление столбцов

Для того чтобы добавить новые поля в табличное представление нужно перейти в resources/js/models/Example.js,

в table.fields добавить объект, где {key: "Ключ объекта", label: "Отображаемое название столбца"}, который приходит с бэкенда

export default {
  table: {
    fields: [
        {
          key: "id",
          label: "#"
        },
        {
          key: 'role',
          label: 'Роль',
          callback: item => {
            return item.role.name
          }
      },
    ],
  }
}

# Настройка столбцов

# Преобразование данных

Если необходимо как-то обработать данные в таблице, выводить что-то кастомное, можно заюзать callback:

{
    key: "name",
    label: "Название",
    callback: model => 'Все что угодно'
}

# Ссылка

Если нужно, чтобы ячейка таблицы была ссылкой, можно передать URL в необязательное поле href:

{
    key: "name",
    label: "Название",
    href: model => `/models?model_id=${model.id}`
}

# Вывод в виде цветного бейджа

Также, значение ячейки таблицы можно оформить в виде badge, используя соответствующее поле. Может принимать значения стандартных Bootstrap цветовых вариантов (primary, danger, info и т.п) или функцию-callback, возвращающую одно из значений:

{
    key: "name",
    label: "Название",
    badge: 'danger', // 1 вариант
    // или
    badge: model => model.count > 2 ? 'success' : 'danger' // 2 вариант
}

# Изображение в ячейке

Значение ячейки таблицы может быть изображением. Использование данного поля с валидным значением src позволяет открыть просмотр изображения в полноэкранном режиме при клике на ячейку. Пример применения:

{
    key: "name",
    label: "Название",
    image: {
        src: model => model.logo_image,
        style: {
            margin: '20px',
            minHeight: '1337px',
            maxHeight: '1337px'
            // ...и любые другие валидные css-свойства из JS
        }   
    
    }   
}

# Ограничение длины

Длина выводимых в ячейку данных может быть ограничена с помощью поля lengthLimit. Может принимать целое число или функцию, возвращающую целое число.

{
    key: "name",
    label: "Название",

    limitLength: 50, // 1 вариант - число
    // или
    limitLength: model => model.limit // 2 вариант - функция-callback
}

# Про совместимость полей...

  • Если указано поле image, поля callback, href, badge, lengthLimit игнорируются.
  • Все необязательные поля callback, href, badge, lengthLimit могут быть использованы независимо друг от друга.
  • Если указаны поля callback и lengthLimit, сначала выполнится callback, затем к полученному значению применится ограничение длины lengthLimit.
